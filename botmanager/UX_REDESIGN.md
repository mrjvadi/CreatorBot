# طراحی حرفه‌ای UX — ربات botmanager

هدف: تجربه‌ی کاربری روان، حرفه‌ای و **درآمدزا**. این سند پیش از پیاده‌سازی است؛
بعد از تأیید تو اجرا می‌شود. هر متنی که اینجا می‌آید نهایتاً در i18n (fa/en) می‌رود.

---

## ۱) اصول پایه

1. **یک کیبورد ماندگار (Reply) فقط برای منوی اصلی** — همیشه در دسترس، کم‌عمق.
2. **بقیه‌ی همه‌چیز Inline** — درجا ادیت می‌شود (پیام جدید پشت پیام جدید نمی‌سازد)،
   مسیر تمیز، دکمه‌ها متناسب با وضعیت.
3. **ناوبری ثابت**: هر صفحه‌ی inline یک ردیف ناوبری دارد: `◀️ بازگشت` و در صورت لزوم `🏠 خانه`.
4. **ساختار ثابت پیام**: خط عنوان (آیکن + بولد) → خط خالی → بدنه → خط خالی → راهنما/CTA.
5. **هر اقدام مخرب، تأیید دارد** (حذف سرویس).
6. **حالت‌های خالی (Empty State)** همیشه یک CTA واضح دارند.
7. **کمترین تعداد تپ تا اقدام اصلی** (ساخت سرویس / شارژ / خرید).

## ۲) واژگان آیکن (یکدست در کل ربات)

سرویس 🤖 · ساخت ➕ · کیف‌پول 💰 · پلن 💎 · تنظیمات ⚙️ · بازگشت ◀️ · خانه 🏠 ·
حذف 🗑 · تأیید ✅ · لغو ✖️ · آمار 📊 · ری‌استارت 🔄 · توقف ⏸ · شروع ▶️ · تست 🧪 ·
تگ 🏷 · سرور 🖥 · موفقیت 🎉 · هشدار ⚠️ · خطا ❌ · در حال انجام ⏳ · زمان ⏰ · پرداخت 💳

وضعیت سرویس (badge رنگی): 🟢 فعال · 🟡 در حال راه‌اندازی · 🔴 متوقف · ⚠️ خطا · ⚫️ نامشخص

## ۳) منوی اصلی (تنها Reply Keyboard)

کاربر:
```
[ 🤖 سرویس‌های من ] [ ➕ ساخت سرویس ]
[ 💰 کیف پول ]      [ 💎 پلن‌ها ]
[ ⚙️ تنظیمات ]
```
ادمین (فشرده، چون بقیه inline می‌شود): `/admin` ورود → پنل inline. Reply ادمین:
```
[ 👥 کاربران ] [ 🤖 سرویس‌ها ]
[ 💎 پلن‌ها ] [ 🖥 سرورها ]
[ 📊 آمار ] [ ⚙️ سیستم ]
[ 🚪 خروج از پنل ]
```

## ۴) آنبوردینگ کاربر جدید (تأثیر مستقیم بر درآمد)

اولین `/start` بدون سرویس/اشتراک → یک پیام گرم با **یک CTA اصلی** و یک ثانویه:
```
👋 خوش آمدی، {نام}!

با CreatorBot در چند ثانیه ربات تلگرام اختصاصی خودت رو بساز.
🆓 شروع رایگان: یک ربات برای همیشه، رایگان.

(دکمه‌ها)
[ 🆓 ساخت ربات رایگان ]        ← CTA اصلی، تک‌دکمه و برجسته
[ 💎 مشاهده پلن‌ها ] [ ❓ چطور کار می‌کند؟ ]
```
نکته: «ساخت رایگان» باید کم‌اصطکاک‌ترین مسیر باشد تا کاربر سریع به ارزش برسد (Aha moment).

## ۵) ویزارد ساخت سرویس (مسیر کلیدی درآمد)

نشانگر مرحله در عنوان: `مرحله ۲ از ۴`. در هر مرحله `◀️ بازگشت`، و `✖️ لغو`.

- **مرحله ۱ — انتخاب سرویس**: گرید ۲ستونه اگر تعداد زیاد بود. هر دکمه نام سرویس از DB.
- **مرحله ۲ — انتخاب تگ/نسخه**: هر تگ + توضیح کوتاه؛ badge `🆕 جدیدترین` روی آخرین تگ.
  (تگ `test` فقط برای ادمین.)
- **مرحله ۳ — انتخاب پلن**: قیمت واضح، badge `🔥 محبوب` روی یک پلن، و **آگاه از موجودی**:
  اگر موجودی کافی است → دکمه‌ی سبز `✅ خرید با X TON`؛ اگر نه → `💳 شارژ و خرید` با مبلغ دقیق کسری.
- **مرحله ۴ — توکن**: راهنمای گام‌به‌گام BotFather + دکمه‌ی `❓ راهنمای دریافت توکن`.
- **کارت تأیید**: خلاصه (سرویس/تگ/پلن/قیمت) + یک CTA اصلی (`✅ پرداخت و ساخت` یا `🆓 ساخت رایگان`).

## ۶) کارت سرویس (در «سرویس‌های من»)

هر سرویس یک کارت:
```
🟢 {نوع سرویس} · {تگ}
📛 {نام کانتینر}
⏰ ۱۲ روز مانده        ← رنگ بر اساس نزدیکی انقضا (🟢>۷ روز، 🟡≤۷، 🔴 منقضی)

[ 📊 آمار ] [ ⚙️ تنظیمات ]
[ 🔄 ری‌استارت ] [ ⏸ توقف ]
[ 🗑 حذف سرویس ]          ← مخرب، ردیف جدا
```
پایین لیست: `[ ➕ ساخت سرویس جدید ]` و `[ 🏠 خانه ]`. اگر سرویس زیاد بود → صفحه‌بندی (Pagination با `‹ ۱/۳ ›`).

## ۷) کیف پول و پلن‌ها (درآمد)

- **کیف پول**: موجودی بزرگ و واضح + مبالغ سریع شارژ:
  `[ 1 TON ] [ 5 TON ] [ 10 TON ] [ ✏️ مبلغ دلخواه ]` و `[ 📜 تاریخچه ]`.
- **پلن‌ها**: نمایش مقایسه‌ای، badge `🔥 محبوب‌ترین`، نمایش «صرفه‌جویی» روی پلن‌های بلندمدت
  (مثلاً «۲ ماه رایگان»)، و خرید تک‌تپ آگاه از موجودی.
- **کمبود موجودی**: یک‌تپ به شارژِ دقیقاً مبلغ کسری (نه بیشتر، نه کمتر) → اصطکاک حداقل.

## ۸) ایده‌های افزایش درآمد

1. **یادآور انقضا** (Scheduled): چند روز قبل از انقضا پیام + دکمه‌ی تک‌تپ `🔄 تمدید`.
2. **محبوب‌ترین پلن** برجسته شود (anchor pricing) تا کاربر به سمت پلن میانی برود.
3. **نمایش صرفه‌جویی** پلن‌های بلندمدت («۲۰٪ ارزان‌تر»).
4. **تمدید/ارتقای تک‌تپ** از داخل کارت سرویس و پیام انقضا.
5. **مسیر رایگان سریع** برای رساندن کاربر به ارزش (سپس upsell در سقف ظرفیت — موجود است).
6. **شفافیت اعتماد**: نمایش وضعیت سرور/سلامت تا کاربر با خیال راحت خرید کند.
7. **معرفی دوستان (Referral)** در فاز بعد: کد دعوت + اعتبار هدیه.
8. **کاهش مراحل ویزارد**: اگر یک سرویس فقط یک تگ دارد، مرحله‌ی تگ خودکار رد شود.

## ۹) جزئیات پیاده‌سازی (فنی)

- یک هلسپر مرکزی برای ردیف ناوبری: `navRow(back, home)` تا یکدستی تضمین شود.
- یک هلسپر `progress(step, total)` برای عنوان ویزارد.
- badgeها از i18n.
- صفحه‌بندی با callbackِ `page:<entity>:<n>`.
- همه‌ی رشته‌ها در i18n (fa/en).

1. لحن متن‌ها: رسمی یا صمیمی؟ (پیشنهاد: صمیمیِ حرفه‌ای)
2. badgeها («🔥 محبوب»، «🆕 جدیدترین») را روی کدام پلن/تگ بگذارم؟ خودکار یا دستی توسط ادمین؟
3. یادآور انقضا و تمدید تک‌تپ را همین فاز پیاده کنم یا فاز جدا؟
