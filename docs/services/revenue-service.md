# revenue-service

## این سرویس چیست
موتور قوانین کمیسیون و واریز نهایی — آخرین حلقه‌ی زنجیره‌ی درآمد (`ads-bot`/`community-service` → این سرویس → `botpay`). هر «درآمد» (`Earning`) که از بالادست می‌آید را طبق `RevenueRule` بین owner و پلتفرم تقسیم و واقعاً از طریق `botpay` واریز می‌کند.

## مسئولیت‌ها
- `ProcessEarning`/`CreateAndProcess` — گرفتن یک `Earning`، تقسیم بر اساس نوع (`RevenueType`)، صدور دو تراکنش (سهم owner + سهم پلتفرم).
- `RevenueRule` قابل‌تنظیم (پیش‌فرض‌ها با `SeedDefaultRules`، قابل ویرایش از `/admin`).
- `PlatformWallet` — کیف‌پول پلتفرم که کمیسیون به آن می‌رود.
- REST API برای ثبت درآمد از بیرون (`POST /earn`) و آمار.

## ارتباطات
- NATS: subscribe `earning.created` (از `ads-bot`/`community-service`).
- HTTP: `/earn` (پشت `authMiddleware`)، `/admin/*` (پشت کلید ادمین واقعی).
- `natspayclient`-مانند اتصال به `botpay` برای واریز واقعی هر دو سهم.

## ایرادها و نکات
- **بحرانی، رفع شد (۲۰۲۶-۰۷-۰۲)**: سه مشکل هم‌زمان در این مسیر پول وجود داشت: (۱) هندلر NATS `earning.created` هیچ auth/اعتبارسنجی مبلغی نداشت — یک کلاینت با دسترسی NATS می‌توانست مستقیم یک `Earning` با مبلغ دلخواه (حتی منفی) بسازد و باعث واریز فوری از `botpay` شود؛ (۲) `Earning.RefID` هیچ چک تکراری‌بودنی نداشت — replay یک رویداد قانونی هم می‌توانست دوباره پرداخت کند؛ (۳) روت REST `POST /earn` هر `X-API-Key` غیرخالی را می‌پذیرفت («هر key معتبر برای earn استفاده می‌شود» — کامنت واقعی در کد قدیم)، نه فقط کلید ادمین واقعی. هر سه رفع شد: `ProcessEarning`/`CreateAndProcess` حالا `TotalNano <= 0` را رد می‌کنند و قبل از ساخت رکورد جدید با یک متد تازه‌اضافه‌شده (`FindEarningByRefID`) چک تکراری‌بودن `ref_id` انجام می‌دهند؛ `/earn` حالا `is_admin` واقعی را الزامی می‌کند.
- **باقی‌مانده، کم‌خطر**: اگر واریز سهم پلتفرم شکست بخورد، کد فعلی همچنان کل earning را «موفق» علامت می‌زند و فقط لاگ می‌کند — یعنی owner پول خودش را می‌گیرد ولی کمیسیون پلتفرم بی‌صدا گم می‌شود، بدون صف retry.
- **باقی‌مانده (بخشی از ضعف عمومی NATS)**: خودِ subject `earning.created` هنوز بدون service-auth است؛ idempotency و اعتبارسنجی مبلغ فقط لایه‌ی دومِ دفاعی است، نه جایگزین احراز هویت واقعی سرویس فرستنده.
