# uploader-bot — مهاجرت از PostgreSQL به MongoDB

این ربات دیگر **هیچ داده‌ای در PostgreSQL نمی‌نویسد**. کل لایه‌ی داده به
MongoDB (از طریق `engine.Mongo`) منتقل شده و ساختار کد به‌صورت ماژولار
(فایل‌به‌فایل برای هر دامنه) سازماندهی شده است.

## چه چیزی تغییر کرد

- `internal/models/` — مدل‌های GORM با ساختارهای سند BSON جایگزین شدند
  (`base/media/user/billing/channel` + کلیدهای تنظیمات). دیگر `gorm` و
  `AllModels()` وجود ندارد. شناسه‌ها رشته‌ی UUID هستند و در `_id` ذخیره می‌شوند.
- `internal/store/` — به‌جای GORM روی `ports.DocumentStore` بازنویسی شد و
  فایل‌به‌فایل تقسیم شد: `store.go، user.go، code.go، file.go، folder.go،
  plan.go، payment.go، channel.go، admin.go، setting.go، stats.go،
  backup.go، download.go، ads.go`. هر کوئری با `instance_id` فیلتر می‌شود
  (multi-tenant) — `store.New(engine.Mongo, engine.InstanceID)`.
- `cmd/bot/main.go` — فراخوانی `eng.DB.Migrate(...)` حذف شد (بدون schema پوستگرس).
- `internal/tgbot/` — هندلرها با شناسه‌های رشته‌ای هماهنگ شدند و فایل
  `admin_panel.go` اضافه شد (نهایی‌سازی آپلود آلبوم، توگل/تنظیم تنظیمات،
  حذف خودکار لینک از کپشن، پیش‌نمایش در کانال، پوشه‌ی جدید، بررسی عضویت).
- `go.mod` — `github.com/google/uuid` و `go.mongodb.org/mongo-driver` به
  requireهای مستقیم اضافه شد.

## نکته درباره‌ی engine

طبق خواسته، `engine` باقی ماند. این ربات چیزی در پوستگرس نمی‌نویسد؛ تنها
خود engine یک اتصال پوستگرس برای **خواندن** متادیتای `bot_instances`
(در `loadInstanceInfo`) نگه می‌دارد. Redis/NATS/heartbeat بدون تغییر است.
اگر می‌خواهی engine اصلاً به پوستگرس وصل نشود، باید `shared-core/engine`
تغییر کند (روی همه‌ی باتها اثر دارد) — این کار انجام نشده تا باتهای دیگر نشکنند.

## کامپایل

در محیط من Go 1.25 (موردنیاز `go.work`) و کش ماژول‌ها در دسترس نبود، پس
نتوانستم اینجا build کنم. لطفاً اجرا کن:

```bash
cd CreatorBotV3
go build ./uploader-bot/...
# در صورت نیاز:
go work sync && go mod tidy -C uploader-bot
```

اگر خطای کامپایلی دیدی، متن خطا را بفرست تا سریع رفع کنم.

## وضعیت امکانات

پایه‌ی داده‌ای و تنظیماتِ همه‌ی امکانات لیست‌شده آماده است (کلیدهای تنظیمات
در `models`، فیلدهای تکی روی `Code` مثل قفل فوروارد/ضدفیلتر/رمز/محدودیت/
آمار فیک، پوشه و زیرپوشه، کانال جوین اجباری و پیش‌نمایش، اشتراک و پرداخت،
ادمین، بکاپ، تبلیغات). هندلرهای پنل برای بخش‌های اصلی سیم‌کشی شده‌اند؛
برخی امکانات پیشرفته (مثل درگاه‌های آنلاین واقعی، کاور ویدیو، بکاپ/ریستور
کامل فایل‌ها) در گام بعد روی همین زیرساخت تکمیل می‌شوند.
