# uploader-bot

ربات آپلودر فایل (فایل → لینک تلگرام) برای رباتساز CreatorBotV3.
هر ربات با توکن خودش `instance_id` یکتا می‌گیرد و داده‌اش روی MongoDB کاملاً ایزوله است.

## اجرا
```bash
go build ./uploader-bot/...
go run ./uploader-bot/cmd/bot
```
تنظیمات اتصال از `.env` خوانده می‌شود (BOT_TOKEN, OWNER_ID, MONGO_URI, REDIS_ADDR, NATS_URL, ...).
این ربات **چیزی در PostgreSQL نمی‌نویسد**؛ همه‌ی داده‌ها در MongoDB است.

## ساختار پکیج
```
internal/
├── models/   مدل‌های سند (BSON)
├── store/    لایه‌ی دیتابیس (Mongo) + کش
├── payment/  درگاه‌های زرین‌پال/زیبال
├── util/     توابع خالص (HTML/entities)
└── tgbot/    هندلرها (callbacks/locks/broadcast/panel/...)
```

## پنل ادمین
با `/start` (ادمین) یا `/panel` باز می‌شود — کاملاً inline و دسته‌بندی‌شده:
آپلود، رسانه‌ها (لیست/اسلایدی)، پوشه‌ها، کاربران، آمار، قفل‌ها، پیش‌نمایش،
اشتراک‌ها، تبلیغات، همگانی، صف تایید، ابزارهای انبوه، بکاپ، ادمین‌ها، تنظیمات،
وضعیت همگانی، روشن/خاموش.

## نکات کلیدی

### قفل‌ها (جوین اجباری/اختیاری)
- حالت اجباری/اختیاری، نوع کانال/گروه/ربات/لینک، حد عضو (غیرفعال‌سازی خودکار).
- گزارش لفت: نیازمند ادمین‌بودن ربات در کانال و فعال‌بودن `chat_member` در allowed_updates پولر.

### کانال ذخیره‌سازی (پایداری در برابر تغییر توکن)
تنظیمات → دسترسی و آپلود → «کانال ذخیره‌سازی». ربات باید در آن کانال ادمین باشد.
فایل‌ها هنگام آپلود به آن کپی می‌شوند و اگر `file_id` بی‌اعتبار شد، از همان‌جا تحویل می‌شوند.

### ارسال همگانی
کپی/فوروارد هر نوع پیام، صف پس‌زمینه با محدودیت نرخ (FloodWait)، حذف خودکار بعد از
X ساعت، و وضعیت زنده با کد (توقف/حذف‌فوری).

### پرداخت
۵ روش قابل‌فعال‌سازی؛ زرین‌پال/زیبال واقعی (request+verify). مرچنت/کارت/ولت در تنظیمات.

### آپدیت از NATS
انتشار روی `uploader.settings.<botID>` با یکی از فرمت‌ها، زنده اعمال می‌شود:
```json
{"key":"welcome_text","value":"..."}
{"settings":{"lbl_search":"🔎 جستجو","welcome_text":"..."}}
```

### شخصی‌سازی
همه‌ی متن‌ها و برچسب دکمه‌ها از تنظیمات قابل‌تغییرند (پنل یا NATS).
امضا از placeholder پشتیبانی می‌کند: `{code}`, `{count}`, `{downloads}`.

## وضعیت
نگاه کنید به `FEATURES_STATUS.md`. تنها ریفکتور باز: شکستن `tgbot` به زیرپکیج‌های
فیچر (اختیاری) که بهتر است مرحله‌ای و با کامپایلر انجام شود.
